## 缘起

前段时间去了敦煌。站在莫高窟前，看到壁画上的胡商、僧侣、使臣——来自完全不同的文明，因为同一条路在同一个洞窟里相遇了。

回来后我想深入了解丝路，发现一个大问题：**知识是零散的、孤立的、埋在百科词条里的。** 我想知道张骞的路线，要点开十几个词条在脑子里自己拼。我想知道造纸术怎么传到欧洲，要翻五个不同页面。

我需要一张图，把所有东西连在一起。所以就自己做了。

## 做了什么

把丝绸之路拆成了 **28 个城市 + 18 个人物 + 19 种物产 + 14 个历史事件**，加上 **159 条关系边**，变成一个结构化的知识图谱。

网站提供三个维度的交互：

### 🗺️ 交互地图

Leaflet + CartoDB 暗色底图，城市标注在真实经纬度上，路线用金色虚线连接。点任何一个城市可以看到详情面板。

### 🕸️ 关系网络图

D3.js 力导向图，所有实体（城市/人物/物产/事件）变成节点，关系变成连线。拖动节点整个网络随之运动。你会发现：

- 张骞 → 汉武帝 → 大宛马 → 苜蓿
- 造纸术 → 怛罗斯之战 → 撒马尔罕 → 巴格达 → 欧洲

每条连线都是一个故事。

### ⏳ 朝代时间轴

底部一条时间轴，从西汉到明代。点击"唐"，地图和列表过滤出唐帝国时期的丝路——长安盛景、玄奘西行、怛罗斯之战。

### 🔄 中英双语

一键切换，所有文案同步翻译。

## 技术栈

纯前端，零框架依赖：

```
Leaflet.js  → 交互地图（支持自定义暗色瓦片）
D3.js v7    → 力导向关系网络图
JSON        → 单一数据源，驱动所有视图
Web Speech API → 旁白配音合成（macOS Meijia 女声）
Canvas + MediaRecorder → 视频自动生成
GitHub Pages → 静态部署，零成本
```

核心设计思路：**不是做一个网站，而是养一个活的数据系统。** 往 JSON 里加一条数据，地图、时间轴、关系图全部同步刷新——做一次数据更新，所有内容形态同步受益。

## 内容自动生成

写了一个 Python 脚本 `scripts/generate-article.py`，可以从同⼀份 JSON 数据一键生成：

- 微信公众号文章
- 知识海报 HTML
- 社交媒体卡片

数据更新 → 所有内容形态自动同步，不用手动改多处。

## 🎬 Canvas 视频自动生成

B站视频也是"生成"出来的——零人工剪辑，纯代码驱动：

### 技术链路

```
JSON 路线数据 → Canvas 渲染 → SpeechSynthesis 配音 → MediaRecorder 录制 → .webm
```

- **墨卡托投影**：经纬度 → Canvas 像素坐标，1920×1080 分辨率
- **动画引擎**：`requestAnimationFrame` + easing 缓动函数，逐段绘制路线，脉冲光点沿路径移动
- **导演模式**：8 条路线自动连播，总时长约 3 分 52 秒，开场卡 + 逐条动画 + 结尾卡
- **配音合成**：Web Speech API（macOS Meijia 女声），纪录片风格脚本 + 停顿节奏 + 语速 160 字/分
- **音视频合并**：`AudioContext` 混合 Canvas 视频流 + 语音旁白 → `MediaRecorder` → VP9 `.webm`
- **实时预览**：CSS transform 缩放适配屏幕，空格键预览，R 键录制

改 JSON 数据 = 改视频内容。整个过程不需要打开任何剪辑软件。`video.html` 独立页面承载全部逻辑，约 400 行 JS。

## 为什么值得做

丝绸之路的内容，大多数是两种极端：要么是干巴巴的百科词条，要么是长篇学术论文。缺少一种直观、交互式的方式让人"看到"整个网络。

把张骞、班超、玄奘、马可波罗放在同一张关系图上，把造纸术、火药、丝绸、佛教的传播路线可视化——这种全局视角是文字给不了的。
