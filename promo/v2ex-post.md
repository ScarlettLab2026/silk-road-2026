# V2EX 分享帖 —— 创意 / 分享创造

**标题：** 我做了一个丝路交互知识图谱 —— 用 JSON 驱动的开源项目，求轻喷

**正文：**

技术栈很简单：纯 HTML/CSS/JS + Leaflet.js + D3.js，数据是一份结构化的 JSON 文件。所有内容从同一份数据渲染。

🐫 在线地址：https://scarlettlab2026.github.io/silk-road-2026
📦 GitHub：https://github.com/ScarlettLab2026/silk-road-2026

## 做了啥

把丝绸之路拆成了 28 个城市、18 个人物、19 种物产、14 个历史事件，加上 159 条关系边。所有数据是开放授权的（CC BY-SA 4.0）。

网站有四个维度：
1. 🗺 交互地图 —— Leaflet + 自定义暗色地图，城市按经纬度标注，路线用虚线连接
2. ⏳ 朝代时间轴 —— 底部一条，按"汉-唐-宋-元-明"筛选
3. 🕸 关系网络图 —— D3.js 力导向图，所有实体之间的关联关系可视化
4. 🎬 传播路径动画 —— 6条路线（丝绸西传/造纸术西传/佛教东传/香料东传/瓷器海上丝路等），脉冲光点沿实际经纬度移动，分段解说+速度控制

支持中英双语一键切换。

## 为什么要做这个

之前出去玩了一趟，在敦煌被震撼到了。回来想深入了解丝路，发现网上的内容要么是零散的百度百科条目，要么是长篇学术论文。没有一种方式能让我直观地看到"张骞的路线"、"长安到罗马经过哪些城市"、"造纸术是怎么传过去的"。

所以干脆自己做了一个。核心思路是：**不是做一个网站，而是养一个活的数据系统**。每次往 JSON 里加一条数据，地图、时间轴、关系图全部同步刷新——做一次数据更新，所有内容形态同步受益。

## 数据怎么组织的

```json
{
  "id": "xuanzang",
  "name": "玄奘",
  "type": "僧侣 / 翻译家",
  "era": "唐",
  "description": "唐代高僧，沿丝路西行至印度取经...",
  "significance": 5
}
```

然后通过 edges 连接：
```json
{"source": "xuanzang", "target": "dunhuang", "relation": "途经"}
{"source": "xuanzang", "target": "xuanzang-journey", "relation": "主角"}
```
